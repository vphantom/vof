# CONTRIBUTING

## Communication

- All our documentation, comments and symbol names are in English.

## Coding Style

- Design principles: KISS, YAGNI, DRY, Fail Fast, Postel's law
- Naming conventions:
  - The more local the scope, the shorter the symbols (i.e. single-letter variable names inside a 5-line function)
  - The more global the scope, the more obvious the symbols (i.e. a full verb-noun combo for a public API method)
- Dependencies: kept to standard libraries, use ubiquitous add-ons as necessary
- Use code formatting tools for languages where they are available

## Priorities

From most to least important:

1. Correctness
2. Readability
3. Performance
4. Feature completeness
